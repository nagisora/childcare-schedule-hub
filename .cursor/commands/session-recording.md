# session-recording

今回の作業はここまでにするので、以下に従い **dev-sessions にセッション記録を残して**。

1. **事前確認**  
   - `docs/dev-sessions/README.md` を軽く確認し、運用ルールを思い出す。  
   - セッション開始時に作成した本日の dev-sessions ファイルがあるか確認（なければ `template-session.md` から新規作成してよい）。

2. **テンプレートに沿って今回のセッション内容を反映**  
   - `docs/05-development-phases.md` のルールと `docs/dev-sessions/template-session.md` に従って、今回のセッションを `docs/dev-sessions/` に記録する。  
   - 特に以下のセクションが埋まっている状態にして：
     - **メタ情報 / 今日のゴール**
     - **コンテキスト・合意事項（AIとの議論メモ）**  
       - どんな壁打ちをし、どの方針で進めたかを2〜5行程度で要約
     - **チェックリスト式実装計画書（このセッション）**  
       - 作業タスクの完了状況を `[ ]` / `[x]` で更新  
       - 必要に応じて「実行プロンプト案」を追記・調整
     - **検証・テスト**  
       - 実際に行った確認内容と結果を反映
     - **実施ログ**
     - **結果とふりかえり / 次回に持ち越すタスク**

3. **フェーズ進捗の確認（必要に応じて）**  
   - 必要があれば `docs/05-development-phases.md` のフェーズ進捗も更新して（完了したフェーズ・次に取り組むフェーズなど）。
