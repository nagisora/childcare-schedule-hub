# 推奨案リファクタ実行計画

## 目的

- Cloud Agent でも詰まりにくい運用（`pnpm` 直実行）へ寄せつつ、ローカル再現性（`mise`）は維持する。
- ドキュメントを「正本中心 + 参照最小化」に整理し、保守コストを下げる。
- 品質系ツールは一気に置換せず、`Biome` を formatter から段階導入する。

## フェーズ1: 実行基盤の方針統一（低リスク・先行）

- 対象:
  - [README.md](README.md)
  - [docs/README.md](docs/README.md)
  - [docs/04-development.md](docs/04-development.md)
  - [package.json](package.json)
- 実施内容:
  - README/開発ガイドのコマンド表記を `pnpm` 主体へ変更（`mise exec -- ...` は補足に格下げ）。
  - `mise` の位置づけを「ランタイムバージョン管理（任意導入）」として明文化。
  - ルート `package.json` に `packageManager` / `engines` を追加して、`mise` 非依存でも期待バージョンを明示。
- 完了条件:
  - `README` と `docs` の記述方針が一致し、`mise` 必須前提の文言が消えている。

## フェーズ2: Biomeをformatterとして段階導入

- 対象:
  - [package.json](package.json)
  - [apps/web/package.json](apps/web/package.json)
  - [biome.json](biome.json)（新規）
  - [README.md](README.md)
  - [docs/04-development.md](docs/04-development.md)
  - （必要に応じて）[apps/web/.eslintrc.json](apps/web/.eslintrc.json)
- 実施内容:
  - `@biomejs/biome` を導入し、まずは `format` / `format:check` のみ運用開始。
  - `lint` は現状どおり `next lint` を維持（ESLint置換は次段階）。
  - 既存コードへ大規模整形を一度に当てず、差分肥大化を避ける（段階適用）。
  - `README` と `docs/04-development` に「Biomeは段階導入中（format先行、lintはESLint継続）」を明記する。
- 完了条件:
  - formatterコマンドが実行可能で、既存 `lint/test/typecheck` フローを壊さない。
  - 利用者がドキュメントを読んで、段階導入フェーズ中であることを即座に把握できる。

## フェーズ3: docs/dev-sessions の安全な縮退

- 対象:
  - [docs/05-00-development-phases.md](docs/05-00-development-phases.md)
  - [docs/05-09-instagram-account-url-coverage.md](docs/05-09-instagram-account-url-coverage.md)
  - [docs/05-10-schedule-url-coverage.md](docs/05-10-schedule-url-coverage.md)
  - [docs/20-deferred-work.md](docs/20-deferred-work.md)
  - [docs/README.md](docs/README.md)
  - [docs/dev-sessions/README.md](docs/dev-sessions/README.md)
  - [docs/dev-sessions/template-session.md](docs/dev-sessions/template-session.md)
  - `docs/dev-sessions/YYYY/**`（過去ログ）
- 実施内容:
  - 先に `05-*` / `20` の dev-sessions 参照を「日付 + Issue/PR参照」中心へ置換。
  - 運用ルールを「今後のセッション記録は必要最小限」に簡素化。
  - テンプレ/定義は残し、過去ログ（`YYYY/**`）を削除または archive 扱いに移行。
- 完了条件:
  - 主要ドキュメントに `dev-sessions/YYYY/...` 直リンクが残らない。
  - 参照導線は `docs/README` で自己完結している。

## フェーズ4: 全体リファクタの初回着手（コード本体）

- 対象（優先3件）:
  - [apps/web/components/FacilitiesTable.tsx](apps/web/components/FacilitiesTable.tsx)
  - [apps/web/lib/instagram-search.ts](apps/web/lib/instagram-search.ts)
  - [apps/web/lib/instagram-schedule-search.ts](apps/web/lib/instagram-schedule-search.ts)
  - [apps/web/hooks/useFavoritesSync.ts](apps/web/hooks/useFavoritesSync.ts)
- 実施内容:
  - `FacilitiesTable` の責務分割（表示・フィルタ・整形ロジック分離）。
  - Instagram URL/施設名正規化ロジックの共通ユーティリティ化。
  - `useFavoritesSync` の責務分離は別PRで着手（影響が広いため段階化）。
- 完了条件:
  - 主要重複ロジックが共通化され、対象テストがグリーン。

## 検証・移行チェック

- コマンド検証（`mise` 未使用前提）:
  - `pnpm --filter web lint`
  - `pnpm --filter web typecheck`
  - `pnpm --filter web test`
  - `pnpm --filter web build`
- 互換確認（必要時）:
  - `mise exec -- pnpm --filter web test` も通ることを確認し、ローカル運用の後方互換を維持。
- ドキュメント検証:
  - `dev-sessions` 直リンクや古い実行手順が残っていないかを検索で確認。
