# セッションとは（このプロジェクト）

このファイルは、このプロジェクトにおける **「セッションとは何か」** の定義だけをまとめたものです。

- セッションファイルの命名規則や運用手順などの詳細は書きません（別ファイルに委譲します）
- AI に「セッションとは」を伝える際は、原則このファイルだけを参照してください

## 定義

- **セッション**: 1回の開発作業を、**計画 → 実施 → 記録**まで完結させる最小単位
- **目安時間**: 15〜60分程度（短時間でも前進するためのタイムボックス）

## 目的

- 短時間の作業でも迷わず前進できるようにする
- AI と協力して開発する際に、意思決定・チェックリスト・実行内容・結果を1箇所に残す
- 次回の再開を容易にする（持ち越しが発生しても「最新の1セッション」を見れば再開できる状態を作る）

## スコープ（セッションに含めるもの）

- **ゴール**（アウトカム）と**完了条件**
- **タスク**: 原則 1〜3 個
  - 各タスクは「1回の AI 指示（プロンプト）で完結する」粒度を目安にする
- **検証・テスト**: 確認方法と期待結果
- **結果とふりかえり**: 何が完了し、何が未完了か
- **次回に持ち越すタスク**: 今回やらない理由と、次回着手条件（前提）

## 非目的（セッションでやらない / 持ち込まない）

- 4個以上のタスクを詰め込む（セッションが破綻しやすい）
- フェーズ計画の全文貼り付け（正本の更新・参照先が不明瞭になるため）
- 「議論だけ」で終わること
  - 議論はセッション内に要点を記録し、次の具体作業（タスク）に接続する
