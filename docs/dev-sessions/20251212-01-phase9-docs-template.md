## 前提・合意事項（事前議論・壁打ちメモ）

- 今日のセッションで前提とする方針:
  - `05-00` はフェーズ全体の索引・正本（目的/完了条件/代表タスク例/詳細リンク）に寄せる
  - フェーズごとの詳細は `docs/05-<フェーズ番号>-*.md` に切り出し、様式をテンプレで統一する
  - dev-sessions は「その日・その回の計画と実行ログ」に限定し、長期的な正本にはしない
  - 見落とし対策は「全文の重複」ではなく「入口に要点＋リンク」を基本にする
- 議論概要:
  - `05-00` から毎回 dev-sessions を起こす運用だと、手順解釈が揺れて統一の流れになりづらい
  - そこで `05-00`（索引/正本）と `05-<phase>`（詳細計画）に責務分割し、セッションはテンプレで進める方式が有効
- 保留中の論点 / 今回は触らないと決めたこと:
  - `05-<phase>` 詳細テンプレの格納場所は `docs/phase-planning/` に決定（必要なら後で移動）
