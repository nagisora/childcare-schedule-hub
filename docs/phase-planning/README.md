# phase-planning ディレクトリ

フェーズ9以降のフェーズについて、詳細な実装計画を `docs/05-<フェーズ番号>-*.md` 形式の専用ファイルに切り出す際に使用するテンプレートとガイドを格納するディレクトリです。

## 含まれるファイル

### `template-phase-detail.md`

フェーズ詳細ファイルのテンプレートです。新しいフェーズ詳細ファイルを作成する際は、このテンプレートをコピーして使用します。

**使い方**:
1. このテンプレートを `docs/05-<フェーズ番号>-*.md` 形式でコピー
2. 各セクションを埋める（`phase-planning.md` のガイドを参照）

**ファイル名例**: `docs/05-10-schedule-url-coverage.md`

### `phase-planning.md`

フェーズ詳細ファイル作成のガイドです。テンプレートの使い方から各セクションの埋め方まで、具体的な手順を説明しています。

**主な内容**:
- 作成手順（対象フェーズの特定から `05-00-development-phases.md` への反映まで）
- 各セクションの埋め方（概要、前提・制約、実装計画、品質チェック、Runbook反映ポイント、リスク・撤退条件・ロールバック、将来の拡張・メモ）
- 進捗チェックリストの追加方法（推奨）
- 参考例（フェーズ9の実装例へのリンク）

## 使い方

### 新しいフェーズ詳細ファイルを作成する場合

1. **ガイドを読む**: `phase-planning.md` を参照して、作成手順を確認
2. **テンプレートをコピー**: `template-phase-detail.md` を `docs/05-<フェーズ番号>-*.md` 形式でコピー
3. **各セクションを埋める**: ガイドに従って各セクションを埋める
4. **`05-00-development-phases.md` に反映**: 該当フェーズセクションに「詳細計画: `<リンク>`」を追加

### 参考例

- **実装例**: `docs/05-09-instagram-account-url-coverage.md`（フェーズ9の実装例）
  - 進捗チェックリストの追加方法
  - タスクへのアンカーリンクの付け方
  - 各セクションの具体的な記載例

## 注意事項

- フェーズ詳細ファイルは「その時点の作業用ラフ」ではなく、**進捗管理の正本**として扱います
- dev-sessions には必要な抜粋やスナップショットだけを貼り、全文を載せる場合は付録扱いとし、正本が詳細ファイルである旨を明記します
- フェーズ計画の変更や決定事項は、詳細ファイルと `05-00-development-phases.md` の両方を更新します
